# 시험 완료 프로세스

## 1. 시험 중지 요청

-   관리 서버로부터 시험 중지 요청 수신
-   상태 저장소 업데이트 (Redis, DB)

## 2. 상태 전파

-   큐 서버를 통한 상태 변경 전파
-   소켓 서버의 브로드캐스팅
-   모든 참여자에게 알림

## 3. 처리 결과

-   수험자/감독관에게 시험 중지 알림
-   시스템 상태 최종 업데이트

```mermaid
flowchart TB
    subgraph AdminServer["관리 서버"]
        direction TB
        AS["관리 서버(AS)"]
    end

    subgraph WebServer["웹 서버"]
        direction TB
        WS["웹 서버(WS)"]
    end

    subgraph Storage["저장소"]
        direction TB
        DB["데이터베이스(DB)"]
        RD["Redis(RD)"]
    end

    subgraph Queue["메시지 큐"]
        direction TB
        QS["큐 서버(QS)"]
    end

    subgraph SocketServer["소켓 서버"]
        direction TB
        SS["소켓 서버(SS)"]
    end

    subgraph Clients["클라이언트"]
        direction TB
        CB["수험자 브라우저"]
        SB["감독관 브라우저"]
    end

    AS --> WS
    WS --> DB
    WS --> RD
    WS --> QS
    QS --> SS
    SS --> CB
    SS --> SB
```
